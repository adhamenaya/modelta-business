## ğŸ”„ Common Workflows

### Starting a New Business Initiative
1. Create proposal in `templates/proposal-template.md`
2. Store in relevant folder (`product-strategy/`, `marketing/`, etc.)
3. Link from quarterly initiatives in `projects/`
4. Track in `analytics-metrics/`

### Creating a New Market Document
1. Use template from `templates/`
2. Place in appropriate folder
3. Link from folder README or parent index
4. Add to version control

### Updating Company Strategy
1. Edit `business-plan/executive-summary.md`
2. Update related documents
3. Notify stakeholders
4. Add to `reports/` for history

### Launching a New Product
1. Reference `product-strategy/product-roadmap.md`
2. Follow `launch-plan/` checklist
3. Track metrics in `analytics-metrics/`
4. Document results in `reports/`

## ğŸ“ Writing Standards

### File Naming
- Use **kebab-case**: `market-analysis.md`
- Use **date prefix** for time-series: `2026-q1-roadmap.md`
- Use **UPPERCASE** for main indices: `README.md`, `INDEX.md`

### Document Structure
- Start with metadata (type, status, owner, date)
- Include executive summary
- Add table of contents for long docs
- Link related documents
- Include version history for important docs

### Review Schedule
- **Weekly**: Projects and initiatives
- **Monthly**: Marketing and sales metrics
- **Quarterly**: Strategy and roadmap
- **Annually**: Comprehensive review
